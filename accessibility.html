ACCESSIBILITY

Readable
by design.

Last updated April 25, 2026 · Version 1.0

Summary

Clausemate is designed so that every core task — scanning a contract, reading the summary, asking a question, comparing two documents, and managing data on the Profile tab — can be completed by users who rely on iOS's built-in accessibility technologies. The list below describes what the app supports today on iPhone. Each entry reflects what we've actually built and tested, not aspirations.

VoiceOver

Every tappable control in Clausemate has an accessibility label. Icon-only buttons — the Ask-tab Send arrow, the Share-as-PDF button on the document detail view, the floating scan button on the Documents tab, the close X on the paywall, the language picker, the "Support Clausemate" tip button — all expose a clear spoken label and, where relevant, an accessibility hint describing what the control does.

Decorative ornaments (the breathing wax-seal mascot, the gold double-rule separators, the parchment textures) are hidden from the accessibility tree so they do not clutter the reading order. Bubble transcripts in the Ask chat are grouped so each turn reads as a single coherent element.

To try it: Settings → Accessibility → VoiceOver → On. Open Clausemate and swipe right to tour the controls.

Voice Control

Because every control exposes a proper accessibility label, iOS's Voice Control surfaces numbered overlays on all of them. Say "Show numbers" after enabling Voice Control and you will see a number on every button on the current screen. "Tap 3" activates the third control; "Swipe up" and "Swipe down" scroll; "Go back" exits modal sheets. No button in Clausemate is reachable only by gesture.

To try it: Settings → Accessibility → Voice Control → Turn on. Then speak commands at a normal volume.

Larger Text (Dynamic Type)

Body copy and most headings scale with the iOS system text size, including the accessibility sizes (200%+). This is handled by a single readable-modifier applied across the app that takes your chosen size and adapts each view's typography. A few display-only elements — the 34-point Garamond hero headlines — stay a fixed size to preserve the editorial layout, but every informational string scales with your preference.

To try it: Settings → Accessibility → Display & Text Size → Larger Text → drag the slider. Return to Clausemate and the body text grows.

Reduced Motion

When Reduce Motion is on, Clausemate suppresses its two non-essential animations: the slow "breathing" rotation on the wax-seal mascot and the gold-dust particle layer on the paywall. Page transitions remain crisp but skip any parallax. You do not lose any information — every visual cue conveyed via motion is duplicated in text or color.

To try it: Settings → Accessibility → Motion → Reduce Motion → On.

Differentiate Without Color Alone

Red flags on clauses are never signaled by color alone. A flagged clause carries a filled burgundy circle, a warning-triangle SF Symbol, and a text badge that reads "Red flag". On the Home tab's sample hero card, the count of clauses to review appears as an inline text pill ("2 clauses to flag"), not just a colored dot. The live privacy pill ("ON-DEVICE") pairs an olive capsule with a lock icon and a bold monospace label.

Sufficient Contrast

Clausemate's primary text is ink on parchment, which measures approximately 11:1 contrast — above WCAG AAA 7:1. Burgundy on parchment measures approximately 7.5:1, also above AAA. Secondary text uses a lightly-reduced ink that remains above the 4.5:1 AA threshold. We use italic Garamond sparingly and never for informational content that must be parsed quickly.

Not supported in v1.0

Dark interface. Clausemate v1.0 runs in light mode only — the parchment aesthetic is the whole product's visual identity. A true dark theme is tracked as a future consideration, not a near-term plan.

Captions and audio descriptions. Clausemate has no video or audio content, so these accessibility technologies do not apply.

Tell us what we missed

Accessibility work is never finished. If you hit a control that lacks a label, a screen that reads poorly with VoiceOver, a view that breaks at the largest text size, or a feature where Voice Control can't reach a button — please email 89leon@gmail.com. Include the iPhone model, iOS version, and which accessibility feature you use. We read every report.
